Development

Debug mode

“Debug mode” outputs the computed JavaScript plot structure (the one passed to Plot.plot).

To activate it, add debug=True when creating your plot generator object:

op = Obsplot(render="widget", debug=True)

If using the widget renderer, debug output will be displayed in JavaScript console. With the jsdom renderer, it will be displayed directly in the notebook.

Install from git

To install pyobsplot from git:

pip install git+https://github.com/juba/pyobsplot.git

To install pyobsplot npm package from git:

npm install 'https://gitpkg.now.sh/juba/pyobsplot/js/pyobsplot-js?main'

Install fom source

If you want to build pyobsplot from source, you’ll have to follow these steps:

  • Install node.js
  • Install poetry dependency management system
  • Clone the repository
  • Install Python dependencies with : poetry install
  • Install main JavaScript dependencies with : npm install
  • Install pyobsplot-js dependencies with : cd js/pyobsplot-js && npm install

pyobsplot widget renderer uses the anywidget Jupyter widget framework. Python source code is in src/pyobsplot/widget.py. JavaScript and CSS source code is in js/widget.

pyobsplot jsdom renderer is provided by an npm package. Source code is in js/pyobsplot-js. Python source code is in src/pyobsplot/jsdom.py.

Widget JavaScript modification and bundling

Each time widget JavaScript or CSS code is modified in the js/widget directory, it must be bundled with one of the following scripts:

# Bundle only once
npm run bundle
# Bundle and watch for code change
npm run watch

The result is put into src/pyobsplot/static.

Tests

pyobsplot comes with three types of tests : Python test, JavaScript tests and Visual regression tests.

To run Python tests, use:

npm run pytest

To run JavaScript tests, use one of those:

# From project root
npm run jstest
# Equivalent to
cd js/pyobsplot-js && npm run test

Visual regression tests allow to test if generated plots stay the same. They use the galata framework. To run them you first have to start a specific Jupyter instance with:

npm run uistart

If you use visual regression tests for the first time, you have to generate reference images with:

# Create / update reference images
npm run uitest:update

After that, you can test that currently generated images are the same as the reference one with:

# Compare with reference images
npm run uitest

Maintenance notes

Releases

  • Release the Python package with poetry build and poetry publish
  • If necessary release the npm package with npm publish in its directory

New Plot versions

When a new Plot version is released:

  • update with npm install @observablehq/plot and npm install d3
  • run tools/get_plot_methods.js and update utils.py
